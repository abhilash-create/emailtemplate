Why fixed logo size?
⚠️
Email clients don't scale reliably
Outlook (desktop) ignores CSS width rules and renders images at their actual file size. A 600px PNG will break the layout entirely — only a fixed width attribute on the <img> tag is respected across all clients.
🔍
HubSpot auto-resize blurs logos
HubSpot compresses images on upload. Letting it scale a large logo down often produces a blurry result, especially on logos with fine text. Uploading at the exact display size — or a clean @2× file — bypasses this entirely.
📱
Retina screens need a 2× file
iPhones, MacBooks, and most Android flagships use high-DPI displays. A 200px logo file looks soft on retina screens. Supply a 400px file but constrain it to 200px in HubSpot's width field to stay crisp everywhere.
🎨
Consistent brand proportions
A logo too tall pushes the headline below the fold on mobile. Too wide, it dominates the header. The 200 × 60px cap keeps the logo proportionate and intentional across every screen size and email client.
🚫
Some clients block images by default
Outlook and many corporate clients hide images until the user clicks "Download images." A fixed-size slot with descriptive alt text (e.g. "Reach Mobile") keeps the layout intact and the brand name readable even when the logo is invisible.
